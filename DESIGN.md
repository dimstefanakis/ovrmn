<section_guidance>

1. Hero

- Create a striking monochrome hero composition.
- Use a memorable visual, not a generic product shot.
- Include a sharp headline, concise subheadline, primary CTA, secondary CTA.
- The hero should feel like entering an intelligence lab or observatory.

Headline territory:

- Tailored AI agents for real systems.
- Intelligence, built to fit.
- Bespoke agents for operational reality.
- AI systems for companies that need more than chat.
- Build machine intelligence around the way you actually work.
- See weak signals. Automate what matters.

2. Positioning / trust strip

- A compact, blunt section under the hero.
- Clearly explain what the company actually delivers.
- Examples:
  - bespoke internal agents
  - knowledge systems
  - workflow automation
  - research copilots
  - decision support
  - human-in-the-loop operations

3. Capabilities

- Present 4–6 strong capability blocks.
- Make them feel intentional and dense, not fluffy feature cards.
- Example capabilities:
  - Internal knowledge agents
  - Support and operations agents
  - Research and intelligence workflows
  - CRM / backoffice automations
  - Executive briefing systems
  - Multi-step agent orchestration

4. How we work

- Show a concise methodology.
- Example flow:
  - Observe
  - Model
  - Deploy
  - Refine
- It should feel like systems engineering meets lab process, not agency copy.

5. Example systems / use cases

- Show 3–4 tailored deployments.
- Example:
  - Customer support triage agent
  - Sales research and enrichment agent
  - Internal knowledge retrieval agent
  - Ops assistant for repetitive workflows
- Frame them as bespoke systems for operational reality.

6. Field Notes / Lab Output / Research Log

- This is critical.
- It should imply the company has a public-facing experimental lab / consumer AI research energy.
- This is the bridge between B2B trust and cultural depth.
- It should feel like ongoing inquiry into interfaces, intelligence, and human-machine workflows.

7. Closing CTA

- Strong final message.
- Reassert the company as a serious builder of tailored intelligence infrastructure.

8. Footer

- Institutional, technical, observatory-like.
- Use metadata labels, archival links, subtle system language.
  </section_guidance>

<technical_requirements>

- Mobile responsive with fluid typography and adaptive layouts
- Smooth scroll behavior
- Semantic HTML5 structure
- Performance optimized, no heavy libraries
- Load Google Fonts for typography

  </technical_requirements>

<motion_design>
Implement motion with restraint and precision.

Page Load:

- Orchestrated reveal sequence (0ms → 200ms → 400ms stagger)

Scroll:

- fade-in-up
- slight parallax only where it adds atmosphere
- subtle, not theatrical

Hover:

- scale transforms used sparingly
- underline animations
- controlled color/opacity transitions
- elegant magnetic or tactile button feel if appropriate

Interactive:

- one distinctive interactive touch is enough
- avoid gimmicks
- the interaction should reinforce the observatory / signal / lab identity

Background:

- extremely subtle ambient motion only
- no floating startup blobs
- no loud gradients
- ambient scan/noise/signal drift is acceptable if tasteful
  </motion_design>
